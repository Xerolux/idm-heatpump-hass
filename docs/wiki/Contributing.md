# Contributing

Wir freuen uns uber Beitrage zu diesem Projekt!

## Wie man beitragen kann

- **Bug Reports**: [Erstelle ein Issue](https://github.com/Xerolux/idm-heatpump-hass/issues/new?template=bug_report.md)
- **Feature Requests**: [Erstelle ein Issue](https://github.com/Xerolux/idm-heatpump-hass/issues/new?template=feature_request.md)
- **Pull Requests**: Fork → Branch → Changes → PR
- **Dokumentation**: Wiki-Seiten verbessern oder neue hinzufugen
- **Ubersetzungen**: Hilfe bei DE/EN Ubersetzungen

## Entwicklungsumgebung einrichten

1. Fork das Repository
2. Klone den Fork lokal
3. Kopiere `custom_components/idm_heatpump_v2/` in dein HA `custom_components/` Verzeichnis
4. Aktiviere Debug-Logging
5. Starte HA neu

## Code-Style

- Verwende [Ruff](https://docs.astral.sh/ruff/) fur Linting und Formatting
- Folge den [Conventional Commits](https://www.conventionalcommits.org/) fur Commit-Messages
- Halte den Code einfach und verstandlich

## Pull Request Prozess

1. Erstelle einen Fork
2. Erstelle einen Feature-Branch (`git checkout -b feat/mein-feature`)
3. Committe deine Anderungen (`git commit -m 'feat: add mein feature'`)
4. Push zum Fork (`git push origin feat/mein-feature`)
5. Erstelle einen Pull Request

## Lizenz

Durch Beitrage stimmst du zu, dass dein Code unter der MIT-Lizenz veroffentlicht wird.
