# Community & Support

GitHub Discussions is the shared space for questions, ideas, examples and
experience reports around IDM Heatpump for Home Assistant. Structured forms
collect the versions and device details that are usually needed for a useful
answer.

## Choose the right place

| Topic | Where | Use it for |
|---|---|---|
| Release news | [Announcements](https://github.com/Xerolux/idm-heatpump-hass/discussions/categories/announcements) | Automatically published stable and pre-release announcements |
| Questions & compatibility | [Q&A](https://github.com/Xerolux/idm-heatpump-hass/discussions/categories/q-a) | Setup help, Navigator/firmware reports, registers and operation |
| Feature requests | [Ideas](https://github.com/Xerolux/idm-heatpump-hass/discussions/categories/ideas) | Discuss the problem, benefit and possible solution before implementation |
| Lovelace & automations | [Show and tell](https://github.com/Xerolux/idm-heatpump-hass/discussions/categories/show-and-tell) | Reusable dashboards, cards, YAML, blueprints and screenshots |
| General exchange | [General](https://github.com/Xerolux/idm-heatpump-hass/discussions/categories/general) | Project-related experience that does not fit another category |
| Reproducible defects | [GitHub Issues](https://github.com/Xerolux/idm-heatpump-hass/issues/new?template=bug_report.md) | Confirmed bugs with steps and diagnostics |
| Security reports | [Security policy](https://github.com/Xerolux/idm-heatpump-hass/security/policy) | Vulnerabilities and sensitive security concerns |

## Before posting

1. Search the [documentation](Home), existing Discussions and Issues.
2. Include the integration, Home Assistant, Navigator and firmware versions.
3. Describe the goal, current result and steps already tried.
4. Add only the relevant log or diagnostic excerpt.
5. Remove IP addresses, serial numbers, web PINs and technician access codes.

For hardware and firmware reports, also include the exact heat-pump model,
connection type and which functions or registers are available. These reports
help improve the [compatibility matrix](Compatibility-Matrix).

## Good community hygiene

- Mark the helpful answer in Q&A when the question is solved.
- Keep one technical question or idea per discussion.
- Move a confirmed, reproducible software defect to an Issue and link both items.
- Share Lovelace and automation examples with their dependencies and sanitized YAML.
